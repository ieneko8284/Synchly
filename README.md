🚀 今後の課題とロードマップ (Roadmap & Issues)
現在、MVP（最小機能製品）として「ユーザー一覧表示」「いいね機能」までを実装済み。今後は以下の改善および機能拡張を予定しています。

1. アーキテクチャの最適化（リファクタリング）
三層アーキテクチャの徹底分離:

現在、Controller内に記述されているSQLクエリを Repository クラスへ完全に移譲し、データアクセス層を独立させる。

ビジネスロジック（マッチング判定など）を Service クラスに集約し、Controllerの責務を「リクエストの制御とレスポンスの返却」に限定する。

フロントエンドのコンポーネント化:

app.js 内でテンプレートリテラルとして記述されているHTML生成ロジックを、再利用可能な関数（コンポーネント）として切り出し、保守性を向上させる。

2. コア機能の拡張
「いいねをくれた人」一覧機能:

自分に興味を持ってくれたユーザーを一覧表示し、そこから「いいね返し」ができる導線を作る。

マッチングロジックの実装:

双方向の「いいね」が成立した際に、ステータスを「マッチング成立」に更新するロジックの実装。

メッセージ（チャット）機能:

マッチング成立者限定のリアルタイムメッセージ交換機能。

3. ユーザー体験（UX）の向上
検索・フィルタリング機能:

性別以外にも、特定の条件（居住地、キーワード等）でユーザーを絞り込める検索窓の実装。

プロフィールの充実:

自己紹介文（bio）の編集機能および、複数枚の画像アップロードへの対応。

ナビゲーションの整理:

マイページ内の重複メニュー（いいねをくれた人）を整理し、ユーザー導線をシンプルにする。

4. 開発環境・運用の高度化
静的解析・テストの導入:

PHPUnitを用いた単体テストの導入。

UI/UXのブラッシュアップ:

インラインCSSを外部スタイルシートへ完全に移行し、共通コンポーネントのデザインガイドラインを作成する。
